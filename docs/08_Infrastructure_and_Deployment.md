# ИНФРАСТРУКТУРА И РАЗВЕРТЫВАНИЕ (DEPLOYMENT)

## Архитектура системы
Shadow Capital Bot — это облачное приложение, использующее экосистему Google Cloud и Firebase для обеспечения отказоустойчивости и масштабируемости.

### 1. Хостинг и вычисления (Compute)
- **Платформа**: Google Cloud Run.
- **Технология**: Docker-контейнер на базе Python 3.11.
- **Режим**: Serverless с постоянным выделением ресурсов (min-instances: 1), что позволяет боту быть всегда в сети и мгновенно отвечать пользователям.

### 2. База данных и Хранение
- **Firestore (Firebase)**: Основное хранилище профилей пользователей, логов и настроек.
- **Google Cloud Storage**: Используется для персистентного хранения локальных файлов данных (монтируется в `/app/data` внутри контейнера).
- **Firebase Hosting**: Обслуживает публичные веб-страницы и лендинги (папка `public`).

### 3. Процесс обновления (CI/CD)
Процесс обновления кода полностью автоматизирован через **GitHub Actions**:

1. **GitHub (Source of Truth)**: Весь код проекта хранится в репозитории на GitHub.
2. **Авто-деплой**: При любом пуше (push) в ветку `main`, GitHub автоматически запускает процесс сборки (`deploy.yml`).
3. **Сборка**: Создается новый Docker-образ, который пушится в Google Artifact Registry.
4. **Развертывание**: Новая версия образа автоматически раскатывается на Google Cloud Run.

> [!IMPORTANT]
> **Вам не нужно вводить `git pull` на сервере вручную.** 
> Как только изменения попадают в GitHub (ветка `main`), система сама обновляет работающего бота в течение 1-2 минут.

### 4. Режим разработки
Для внесения правок:
1. Выполните изменения в коде.
2. Сделайте `git commit` и `git push origin main`.
3. Система сама обновит "живую" версию бота.
